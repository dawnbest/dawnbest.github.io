Phlebo is a test-indexing application developed for hospital laboratory facilities. It is a search engine for specimen collection that can be tailored to the exact specifications of the laboratory.

Phlebo is PHP-driven and runs on a MySQL database. It can be modified to feature a login function and offline mode.

Objectives

Problem: Hospital staff within and outside the laboratory are often called on to draw blood and collect other specimens for laboratory testing. In several facilities within the hospital, specimen collection requirement materials were out-of-date and/or static paper documents. Collecting a specimen correctly differs from test to test and errors can cause delays in diagnosis and patient care. Due to the speed in which advances in testing media and best-practices are developed, the documentation is in constant need of updating. Amending specimen requirement documents and distributing them via printout is time-consuming, costly, and inefficient.

Solution: Phlebo takes the guesswork out of specimen collection by allowing hospital staff to quickly look-up a test by name and receive a consolidated set of collection instructions. The central MySQL database is responsive and easy to update, allowing changes to be made instantly and in real-time. Phlebo can be accessed via a website or in an offline mode, on both desktop and mobile devices.
